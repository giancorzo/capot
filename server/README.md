# Capot Server

## Environmant vars

* `PORT`: Port to start server on.
* `WWW`: Path to directory to be served by the web server.
* `DATA`: Path to data directory (if using built-in PouchDB server)
* `DEBUG`: Switch on verbose logging and long stack traces.
* `COUCHDB_URL`: ...
* `COUCHDB_USER`: ...
* `COUCHDB_PASS`: ...
